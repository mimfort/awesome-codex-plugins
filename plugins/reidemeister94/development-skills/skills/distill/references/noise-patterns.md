# Semantic Noise Patterns

LLM-judgment categories the regex pre-pass can't handle. Mechanical kill-lists (hedges, fillers, verbose constructions, slop score) live in `deterministic-patterns.md`.

## Buzzwords — replace with specifics

`delve` -> examine or state the finding; `leverage`/`utilize` -> use; `facilitate` -> help/enable; `tapestry`/`landscape`/`realm`/`paradigm` -> be specific; `cutting-edge`/`state-of-the-art`/`revolutionary` -> name the advance; `holistic`/`comprehensive`/`robust` -> what does it cover; `synergy`/`empower` -> what happens; `seamlessly`/`effortlessly` -> remove; `innovative`/`novel` -> state what's new. If removable without loss, remove.

## Structural padding

Intro paragraphs without information; 5-paragraph structure forced on content that doesn't need it; a header for a single paragraph it repeats; bullets where one sentence suffices; 2-3 row tables clearer as prose; decorative bold/rules/emoji. Structure serves navigation only.

## Non-committal language

`X can be Y` when you mean `X is Y`; `may help` when evidence shows it does; "depends on various factors" without naming them; presenting all sides as equal when evidence favors one; excessive might/could/possibly/potentially. Make the strongest claim the evidence supports; qualify only when the qualification carries information.

## Repetition

Same idea reworded within a section; `X, which means Y` where Y is obvious from X; echo sentences ("This is important. The importance cannot be overstated."); overlapping list items; re-explaining a covered concept. State each fact once, where most relevant.

## Empty transitions

Words that add no contrast or information: "On the other hand" when not actually contrasting; "In terms of...", "When it comes to...", "With respect to...", "In the context of...". Cut, or rewrite to the bare claim.
