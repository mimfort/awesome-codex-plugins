# Radix primitives & shadcn

Demand-loaded. Read this file when the project uses Radix primitives (Dialog, Popover, DropdownMenu, Collapsible, Accordion, Tooltip, etc.) or ships a `components/ui/` directory generated by shadcn. Detection signals: `@radix-ui/*` in `package.json`, presence of `components/ui/`, or a `components.json` shadcn config at the project root.

---

## Default to uncontrolled state

For Radix primitives, let the primitive manage `open` / `onOpenChange` itself. The trigger lives inside the primitive's tree:

```tsx
// Good — uncontrolled, Radix manages open/close
<Dialog>
  <DialogTrigger asChild>
    <Button>Edit</Button>
  </DialogTrigger>
  <DialogContent>...</DialogContent>
</Dialog>
```

Switch to **controlled** only when:

- You need to close programmatically after an async op (e.g. close on API success)
- The trigger is outside the primitive's tree (a row double-click, a keyboard shortcut, a menu item)
- The state coordinates with another component (closing on a sibling open)

When controlled, the standard `onOpenChange` guard:

```tsx
<Dialog open={open} onOpenChange={(isOpen) => !isOpen && onClose()}>
```

This ensures `onClose` fires only on close, not on every open/close transition. Use it verbatim — it's the codebase pattern.

---

## `shadcn add` overwrites local edits

`npx shadcn add` (and its successors) overwrites existing files in `components/ui/`. Always `git diff` after running it and restore any custom modifications. If you customise a generated component, mark the change with a comment so future runs can be reviewed quickly.

---

## What NOT to Do

| Anti-Pattern | Instead |
|--------------|---------|
| Always-controlled Radix state | Uncontrolled by default; controlled only when needed |
| Controlled `onOpenChange` without the close-only guard | `onOpenChange={(isOpen) => !isOpen && onClose()}` |
| Custom edits in `components/ui/` lost to `shadcn add` | `git diff` after every `shadcn add` and restore customisations |
