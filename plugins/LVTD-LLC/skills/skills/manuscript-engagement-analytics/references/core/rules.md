# Manuscript Engagement Analytics Rules

Use these rules when auditing manuscripts for value pacing and reader engagement.

## Core Rules

### 1. Anchor Metrics To The Promise

Word counts only matter relative to reader value.

- Identify the reader and promise before judging length.
- Define what counts as a value event for this book.
- Do not penalize long sections that deliver dense value.

### 2. Measure The Structure First

Create a heading and word-count map before giving structural advice.

- Include line numbers when possible.
- Include cumulative words to estimate reading time.
- Use section-level counts rather than whole-chapter guesses.

### 3. Flag Slow Starts

Readers should receive meaningful payoff early.

- Inspect front matter, introductions, author bio, theory, and background.
- Move, compress, or delete anything delaying the first payoff.
- Keep only the credibility and context needed to make early value land.

### 4. Flag Long Slogs

Long sections need strong value or subdivision.

- Split sections that contain multiple takeaways.
- Rename vague topic headings around reader gains.
- Convert background into examples, diagnostics, checklists, or exercises.
- Move optional depth to companion material or later sections.

### 5. Use Beta Data As A Heatmap

Reader comments and abandonment are behavioral signals.

- Mark where comments cluster.
- Mark where comments stop.
- Compare dropoff points across readers.
- Inspect the preceding section as well as the section where dropoff appears.

### 6. Recommend Structural Fixes Before Prose Polish

Engagement problems are usually structural.

- Move value earlier.
- Cut low-value setup.
- Add examples or checkpoints.
- Reorder sections around reader progress.
- Polish sentences only after the value path works.

## Guidelines

- Treat 250 words as roughly one minute of reading for rough estimates.
- Use thresholds as prompts to inspect, not hard rules.
- For technical books, count a working lab, verification checkpoint, or troubleshooting table as a value event.
- For advanced readers, theory may be valuable earlier if it directly supports their current decision.

## Exceptions

- **Reference books**: Some sections may be intentionally standalone and uneven in length.
- **Narrative nonfiction**: Atmosphere and story can be value, but the reader promise still matters.
- **Academic books**: Longer theoretical setup may be acceptable if the reader expects it and the payoff is clear.

## Quick Reference

| Rule | Summary |
|------|---------|
| Promise first | Metrics depend on reader payoff. |
| Structure map | Measure headings and words before advice. |
| Slow start | Inspect words before first meaningful value. |
| Long slog | Split, move, cut, or add payoff. |
| Beta heatmap | Use comments and dropoff as signals. |
| Structure before prose | Fix value path before polish. |
