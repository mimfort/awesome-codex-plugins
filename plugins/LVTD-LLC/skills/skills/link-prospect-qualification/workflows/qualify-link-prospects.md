# Qualify Link Prospects Workflow

Use this workflow to score and segment raw link prospect lists.

## When to Use

- The user has URLs, domains, or backlink exports to qualify.
- Outreach should not start until prospects are scored.
- A campaign needs reject reasons and priority bands.

## Prerequisites

- Target asset URL and topic.
- Campaign audience and opportunity type.
- Raw prospect list with source evidence when available.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Confirm Campaign Context

- [ ] Record asset URL, topic, market, audience, geography, language, and opportunity type.
- [ ] Identify target segments: resource page, blog, journalist, directory, partner, community, review, interview, or data story.

### Step 2: Check Asset Readiness

- [ ] Confirm the asset is live, indexable, navigable, and shareable.
- [ ] Check internal links to and from related pages.
- [ ] Note missing evidence, methodology, visuals, or conversion path if relevant.

### Step 3: Normalize And Triage

- [ ] Deduplicate URLs and domains.
- [ ] Preserve source query, source list, backlink source, and page type.
- [ ] Remove obvious non-prospects.
- [ ] Use automated signals to sort into manual review, low priority, and reject buckets.

### Step 4: Manually Review Shortlist

- [ ] Check relevance, page purpose, outbound-link motivation, recency, design, ads, text quality, contact path, and community/editorial signals.
- [ ] Identify the person or institution controlling the page.
- [ ] Look for spam or manipulation signals.

### Step 5: Score And Segment

- [ ] Apply hard reject gates first.
- [ ] Score remaining prospects with the rubric.
- [ ] Assign priority, standard, nurture, manual review, or reject.
- [ ] Capture pitch angle or missing check for each kept prospect.

## Common Mistakes

| Mistake | Why It's Bad | Do Instead |
|---------|--------------|------------|
| Sorting by authority only | Ignores relevance and intent | Apply hard gates and weighted rubric |
| Keeping prospects without evidence | Slows outreach and causes spam | Record qualification reason |
| Ignoring asset readiness | Good links point to weak pages | Fix asset first |
| Not tracking rejects | Future research repeats mistakes | Keep reject reasons |

## Exit Criteria

Task is complete when:

- [ ] Every prospect has a decision or clear missing check.
- [ ] Priority prospects have a pitch angle.
- [ ] Rejections include evidence.
- [ ] Next handoff to outreach is segmented.
