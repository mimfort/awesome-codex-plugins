# Run Link Outreach Workflow

Use this workflow to draft, review, and operate link outreach.

## When to Use

- The user needs outreach copy or subject lines.
- A qualified prospect segment is ready for outreach.
- Replies, follow-ups, or tracking need a process.
- A scaled outreach batch needs guardrails.

## Prerequisites

- Qualified prospect segment.
- Linkable asset and destination URL.
- Campaign type and request type.
- Contact data and suppression list when available.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Define Campaign Type

- [ ] Identify request type: resource suggestion, guest contribution, broken-link replacement, attribution, digital PR, expert survey, submission listing, or bulk offer.
- [ ] Record asset value, proof points, audience, and exact destination URL.

### Step 2: Segment Prospects

- [ ] Group by site type, audience, authority, prior relationship, and motivation.
- [ ] Reserve high-value targets for high-touch outreach.
- [ ] Confirm opt-outs and do-not-contact constraints.

### Step 3: Gather Context

- [ ] Capture relevant article, page gap, beat, broken URL, unlinked mention, prior share, or relationship note.
- [ ] Tie the context directly to the ask.

### Step 4: Draft And Review

- [ ] Choose an honest subject line.
- [ ] Draft short outreach with exact URLs, audience benefit, request, and next step.
- [ ] Score against the review rubric.
- [ ] Revise any zero-score criterion before sending.

### Step 5: Pilot, Send, And Track

- [ ] For scaled sends, pilot 3 emails or 5-10 percent of the list.
- [ ] Review replies, bounces, and errors before expanding.
- [ ] Track every send, response, placement, and opt-out.

### Step 6: Handle Responses

- [ ] Categorize replies as ignore, accept, decline, or counter.
- [ ] Send one custom follow-up after silence.
- [ ] Verify live placements and preserve relationship notes.

## Common Mistakes

| Mistake | Why It's Bad | Do Instead |
|---------|--------------|------------|
| Fake personalization | Signals spam | Use context tied to the request |
| Overly broad batch | Low relevance | Segment before templating |
| Deceptive subject line | Breaks trust and compliance | Set accurate expectations |
| Missing exact URLs | Recipient must guess | Include target and destination URLs |

## Exit Criteria

Task is complete when:

- [ ] Outreach is specific, honest, and audience-relevant.
- [ ] Tracking fields and response categories are ready.
- [ ] Pilot or send plan includes opt-out and compliance handling.
