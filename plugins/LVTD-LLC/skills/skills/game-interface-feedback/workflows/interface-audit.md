# Interface Audit Workflow

## Procedure

1. Pick a specific player task.
2. Trace the full interaction map from intention to next decision.
3. Mark every visible and invisible state the player must understand.
4. Identify the current feedback channel for each state.
5. Add missing immediate, consequence, and recovery feedback.
6. Check for accessibility alternatives.
7. Convert the audit into implementation tasks.

## Output Template

```markdown
## Task

## Interaction Map
| Step | Player Intention | Input/System State | Feedback | Risk |

## Feedback Spec
| Event | Channel | Timing | Priority | Accessibility Alternative |

## UI/Control Changes

## Implementation Tasks

## Test Cases
```
