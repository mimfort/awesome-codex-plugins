# Guest Post Placement

Use this skill to plan guest post campaigns where off-site bylined content must serve the publisher's audience, justify its citations, and support an on-site asset or business goal.

This skill is derived from Garrett French and Eric Ward's *Ultimate Guide to Link Building* and uses transformed guidance with durable book-topic references. Do not copy book prose into user outputs.

## Quick Start

1. Use the Contents table below to choose the smallest useful reference set.
2. Use `workflows/plan-guest-post-placement.md` for a full campaign plan.
3. Confirm the target page has a natural citation reason.
4. Translate keywords into audience names before prospecting.
5. Return viability, publisher quality, content titles, outreach constraints, tracking fields, and risks.

## Default Output

When asked for guest post placement help, return:

1. **Campaign goal** - target page, business goal, expertise signal, and risk constraints.
2. **Audience and citability** - keywords, audience names, citable targets, and missing elements.
3. **Viability and prospects** - query signals, opportunity quality, and expertise needs.
4. **Content and outreach plan** - title ideas, niche focus, audience benefit, citation target, and status.
5. **Tracking and risks** - prospect fields, content status, approvals, placement reporting, and common failure modes.

## Core Posture

- Guest posting is editorial contribution, not manufactured link placement.
- Every link needs a reader-serving citation reason.
- Commercial pages often need citable elements before outreach.
- Pitch titles before complete articles in scaled campaigns.
- Track content inventory carefully to avoid duplicate placement.
