# Run Reader Conversations Workflow

Use this workflow to run a small conversation round that improves a nonfiction book's reader, promise, scope, or TOC.

## When To Use

- A book idea or promise feels broad, vague, or untested.
- The author is unsure who the book is really for.
- A TOC exists but has not been tested with real readers.
- A chapter should be taught live before drafting more.

## Prerequisites

- Current reader hypothesis.
- Current promise or TOC.
- Access to at least a few likely readers.

**Reference**: `references/core/rules.md`

---

## Workflow Steps

### Step 1: Define The Round

**Goal**: Know exactly what this batch should teach.

- [ ] Choose listening or teaching mode.
- [ ] Write the primary learning goal.
- [ ] Name the target reader segment.
- [ ] Define the decision the author will make after the round.

---

### Step 2: Recruit Readers

**Goal**: Find readers whose signals matter.

- [ ] List friendly first contacts.
- [ ] Add relevant community, customer, student, or peer contacts.
- [ ] Prioritize people recently facing the problem.
- [ ] Send a low-pressure invitation.

**Reference**: `references/core/examples.md`

---

### Step 3: Prepare The Guide

**Goal**: Ask questions that produce usable evidence.

- [ ] Start with recent lived experience.
- [ ] Ask about failed alternatives and search behavior.
- [ ] Ask for confusing, frustrating, or risky moments.
- [ ] In teaching mode, prepare one narrow TOC slice.
- [ ] Prepare a note schema for language, objections, and book decisions.

---

### Step 4: Run The Conversations

**Goal**: Learn without defending the idea.

- [ ] Keep the reader talking about concrete events.
- [ ] Capture exact phrases.
- [ ] Mark moments of energy, confusion, skepticism, or boredom.
- [ ] Do not argue, explain away, or fish for praise.
- [ ] Ask permission for follow-up or beta reading only after the useful part is complete.

---

### Step 5: Synthesize Signals

**Goal**: Convert notes into manuscript decisions.

- [ ] Group findings by reader segment.
- [ ] Extract repeated reader language.
- [ ] Identify scope cuts, additions, and assumptions.
- [ ] Update promise, TOC, examples, objections, or beta-reader plan.
- [ ] Decide the next test.

## Exit Criteria

Task is complete when:

- [ ] The author knows what changed in the book plan.
- [ ] The next manuscript or testing action is clear.
- [ ] Conversation notes have produced decisions, not just summaries.
