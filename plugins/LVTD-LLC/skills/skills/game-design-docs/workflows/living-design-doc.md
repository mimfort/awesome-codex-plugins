# Living Design Doc Workflow

## Procedure

1. Choose the doc type.
2. Write the stable intent first.
3. Add constraints and explicit non-goals.
4. Add implementation contract:
   - data
   - state
   - UI
   - assets
   - tests
   - telemetry
5. Add open questions and the evidence needed to answer them.
6. Add maintenance triggers.

## Output Template

```markdown
## Purpose

## Current Intent

## Scope

## Non-Goals

## Implementation Contract

## Acceptance Criteria

## Tests And Telemetry

## Open Questions

## Decision Log
```
