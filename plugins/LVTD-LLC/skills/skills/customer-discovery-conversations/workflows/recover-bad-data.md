# Recover Bad Data

Use this workflow when a conversation has drifted into compliments, generic
claims, future hypotheticals, or feature ideas.

## Compliments

- [ ] Do not record praise as evidence.
- [ ] Acknowledge briefly if needed.
- [ ] Return to current behavior, costs, workarounds, or commitments.

## Generic Or Future Claims

- [ ] Ask when it last happened.
- [ ] Ask them to walk through that instance.
- [ ] Ask what they tried, paid for, ignored, delegated, or worked around.
- [ ] Treat future promises as weak until backed by action.

## Feature Ideas

- [ ] Capture the request without obeying it.
- [ ] Ask what it would let them do.
- [ ] Ask how they cope without it today.
- [ ] Decide whether it is a buying criterion, a workaround clue, or noise.

## Recovery Output

For each weak signal, record: `Weak signal`, `Why weak`, `Recovery question`,
and `Usable evidence found`.

## Exit Criteria

The conversation returns to concrete facts, a real commitment, or a clear
negative signal.
