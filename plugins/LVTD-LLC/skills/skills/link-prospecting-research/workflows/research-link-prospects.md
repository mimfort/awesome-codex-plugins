# Research Link Prospects Workflow

Use this workflow to produce a link prospecting research map.

## When to Use

- The user needs link prospect sources or query patterns.
- A linkable asset exists and needs opportunity research.
- Competitor backlinks need to be translated into opportunity types.
- The user wants a research map before qualification or outreach.

## Prerequisites

- Target site and asset.
- Market, geography, language, and audience.
- Competitors if known.
- Excluded tactics or risk constraints.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Define Scope

- [ ] Record site, target asset, audience, geography, language, competitors, and exclusions.
- [ ] Identify assets available for promotion.

### Step 2: Map Assets To Opportunity Types

- [ ] List opportunity types that fit each asset.
- [ ] Record why each opportunity type would care.
- [ ] Reject opportunity types with no asset fit.

### Step 3: Build Keyword Classes

- [ ] Build MDKW, CDKW, PCKW, ITL, CCN, GKW, and RVKW lists.
- [ ] Include synonyms, practitioner labels, and adjacent verticals.

### Step 4: Build Query Library

- [ ] Combine research phrases, operators, and footprints.
- [ ] Add exclusions for recurring noise.
- [ ] Save useful footprints from strong prospects.

### Step 5: Add Competitor And List Sources

- [ ] Review competitor backlink patterns for page type and asset fit.
- [ ] Find list-scrape sources and record filters.
- [ ] Expand seeds with autocomplete where useful.

### Step 6: Prioritize Research Outputs

- [ ] Group opportunities by relevance, apparent volume, uniqueness, effort, and asset fit.
- [ ] Mark unknowns for qualification.
- [ ] Hand off candidate lists to `link-prospect-qualification`.

## Common Mistakes

| Mistake | Why It's Bad | Do Instead |
|---------|--------------|------------|
| Producing generic lists | No reason to link | Start from asset-opportunity fit |
| Digging deep into weak SERPs | Low return on review time | Create more precise query segments |
| Treating backlinks as prospects | Context is unknown | Categorize and qualify later |
| Blending opportunity types | Value assumptions differ | Keep page types separate |

## Exit Criteria

Task is complete when:

- [ ] Candidate sources are grouped by opportunity type.
- [ ] Query patterns and source evidence are preserved.
- [ ] Obvious noise and duplicates are removed.
- [ ] Next qualification checks are clear.
