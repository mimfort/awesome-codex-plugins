# Frame Warm Intro

Use this workflow to draft a credible request for a customer conversation.

## Steps

- [ ] Name the specific segment or person type.
- [ ] Choose the warmest access path available.
- [ ] State the broad vision without pitching the solution.
- [ ] Explain the current stage honestly.
- [ ] Admit the specific thing the team is trying to understand.
- [ ] Explain why this person is unusually helpful.
- [ ] Make a short, concrete ask.
- [ ] Keep the request easy to forward.
- [ ] Remove product claims, decks, demos, and broad "pick your brain" phrasing.

## Draft Shape

```markdown
Hi [Name],

I am working on [broad problem or vision]. We are at [stage] and are trying to
understand [specific unknown].

You have experience with [reason they can help], so your perspective would help
us avoid guessing.

Would you be open to a 15-20 minute chat next week?
```

## Output

Return the final message plus a one-sentence forwardable intro if someone else
needs to make the introduction.

## Exit Criteria

The message sounds like a focused request for help, not a disguised demo, survey,
or pitch.
