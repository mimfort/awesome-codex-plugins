# Run Broken Link Building Workflow

Use this workflow to run broken link building or owned-site link equity salvage.

## When to Use

- The user wants broken link prospects.
- The user has a replacement resource.
- The user wants to recover lost backlinks after a migration, redesign, or deleted content.

## Prerequisites

- Target asset or owned domain.
- Topic and audience.
- Access to crawling, status checking, or backlink data when available.

**Reference**: `references/core/knowledge.md`, `references/core/rules.md`

## Workflow Steps

### Step 1: Define Asset Or Salvage Scope

- [ ] For BLB, record replacement URL, topic, audience, and replacement strength.
- [ ] For salvage, record owned domain, migration/redesign history, and URL sources to inspect.

### Step 2: Find Resource Pages

- [ ] Search topical resource, link, reference, FAQ, directory, and list pages.
- [ ] Keep pages where the asset would fit with light scrutiny.
- [ ] Track source query and page type.

### Step 3: Scrape And Check Outbound Links

- [ ] Extract outbound URLs from candidate pages.
- [ ] Track source page, outbound URL, anchor text, and context.
- [ ] Bulk check status.
- [ ] Recheck every dead or unclear result.

### Step 4: Prioritize Dead URLs

- [ ] Gather referring domains or inbound link metrics for verified dead URLs.
- [ ] Review Archive.org, anchor text, and page title to understand old content.
- [ ] Classify dead type, topic, scope, and replacement fit.

### Step 5: Pull And Qualify Dead Backlinks

- [ ] Export backlinks for worthwhile dead URLs.
- [ ] Filter toward resource-style referring pages.
- [ ] Deduplicate to one selected URL per domain.
- [ ] Confirm referring page loads and still links to the dead URL.
- [ ] Choose best contact route.

### Step 6: Prepare Outreach Or Salvage

- [ ] For outreach, prepare exact page, dead link, replacement URL, fit note, and disclosure.
- [ ] For salvage, map old URL to closest live destination and verify redirect.

## Common Mistakes

| Mistake | Why It's Bad | Do Instead |
|---------|--------------|------------|
| One status check | False positives | Recheck dead URLs |
| Raw backlink export outreach | Wastes time | Filter, dedupe, verify |
| Weak replacement | Low conversion and spam risk | Create or improve asset |
| Homepage redirects | Poor user fit | Map to closest relevant destination |

## Exit Criteria

Task is complete when:

- [ ] Dead URLs are verified.
- [ ] Referring pages are qualified and still contain the dead link.
- [ ] Replacement fit is documented.
- [ ] Outreach notes or redirect map are ready.
