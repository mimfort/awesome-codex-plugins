# Evergreen Technical Book Checklist

Use this checklist to audit a technical manuscript for durability.

## Promise And Scope

- [ ] The promise is valuable beyond one tool cycle.
- [ ] The target reader is clear.
- [ ] The book says what it will not cover.
- [ ] Tool choice supports the promise instead of becoming the promise.

## Content Half-Life

- [ ] Durable concepts are identified.
- [ ] Semi-durable workflows and examples are identified.
- [ ] Volatile details are identified.
- [ ] Screenshots, UI steps, prices, vendor policies, and versions are flagged.

## Book vs Companion

- [ ] Core model stays in the book.
- [ ] Reader safety warnings stay in the book.
- [ ] Volatile walkthroughs move to companion resources where useful.
- [ ] Companion resources do not withhold the promised value.
- [ ] Book references to companion resources are stable.

## Update Policy

- [ ] Companion home is defined.
- [ ] Owner is defined.
- [ ] Review cadence is defined.
- [ ] Update triggers are defined.
- [ ] Last-reviewed or version markers are planned.
- [ ] Reader issue/report path exists.

## Reader Experience

- [ ] Moving content out does not make the book too abstract.
- [ ] Examples remain concrete enough to understand.
- [ ] The reader knows when current details may differ.
- [ ] The reader can still complete the core learning path.

## Red Flags

- The manuscript contains many UI screenshots for fast-changing tools.
- Exact prices or policy details appear without date or source.
- Tool walkthroughs bury the durable lesson.
- Companion links are scattered and fragile.
- The book cannot deliver value without a website login or upsell.

## Quality Rubric

| Dimension | Strong | Weak |
|-----------|--------|------|
| Durability | Teaches lasting capability | Teaches one UI flow |
| Specificity | Concrete example plus model | Abstract advice only |
| Split | Volatile details updateable | Fragile details embedded |
| Completeness | Book delivers promise | Companion required for core value |
| Maintenance | Update policy exists | Drift discovered by readers |
